WHAT YOU GET

这次正式版修复了什么

本次发布重点不是加新花样，而是把已有承诺彻底对齐，避免文档说一套、示例做一套、skill 又引导另一套。

三模式统一

正式去掉 Deck 相关残留描述，所有入口都只保留 Editorial、Dashboard、Single。

模板纠偏

修复 design-system 中 Dashboard 骨架误写成 Editorial 的问题，避免跟着模板做错页面。

版本同步

示例、下载命令、文案源、页脚与 changelog 统一到当前正式版版本号。

顶部视觉图

官方介绍页新增顶部横幅，并把图里的旧版本徽标替换为当前版本。

THREE MODES

当前只保留三种正式布局模式

展示感不再依赖翻页动画，而是通过章节节奏、卡片密度和首屏构图来实现。

A
Editorial

长滚动叙事

适合分析报告、方案展示、产品介绍。通过章节分隔和卡片层次制造连续阅读节奏。

B
Dashboard

高密度看板

适合数据汇总、状态跟踪、多维对比。一屏给出全貌，重点信息用指标卡和状态条承载。

C
Single

单屏聚焦

适合活动海报、单页简介、邀请函。信息量小，但要求首屏足够完整且有冲击力。

RELEASE RULES

正式版输出的硬约束

以下规则来自当前 skill 与验收清单，示例页本身也按这些限制实现。

必须满足

  • 单文件 HTML，样式全部内联在 <style> 中
  • 默认可见、无需点击、无需等待，打印预览即最终呈现
  • 只依赖 Google Fonts 和 Font Awesome CDN
  • 桌面、平板、手机都有基础响应式回落

明确禁止

  • 禁止 animation、transition、scroll-snap、hover 动效
  • 禁止 JavaScript 交互逻辑和默认隐藏内容
  • 禁止 Inter、Roboto、Arial 等通用字体作为正式输出主字体
  • 禁止把内容堆成普通文档排版，必须有卡片和节奏
INSTALL

安装与使用

正式版 release 会附带更新后的 html-beautifier.skill 资产，安装后即可直接调用。

下载并安装

mkdir -p ~/.claude/skills curl -L https://github.com/HeiGeAi/html-beautifier/releases/download/v1.2.2/html-beautifier.skill \ -o ~/.claude/skills/html-beautifier.skill

开始使用

/html-beautifier 把这个报告做成网页 场景：数据报告 风格：商务专业 模式：Dashboard
