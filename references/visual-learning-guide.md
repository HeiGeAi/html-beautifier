# 视觉学习指南（参考网页学习流程）

当用户提供参考网页时，使用本指南进行视觉拆解和创新设计。

## 核心原则

**学精髓，不抄结构**

目标不是复制一个壳，而是把参考页的节奏、卡片、层级、留白、系统感转译成新的页面。

## 第一步：参考页采集

### 1.1 获取参考内容

- 如果是线上 URL：使用 WebFetch 工具获取页面内容
- 如果是本地 HTML：直接读取文件
- 如果两者都有：同时检查，确保获取真实内容（注意 iframe 嵌套）

### 1.2 区分页面类型

必须区分参考页是：
- 首页总览
- 内容报告页
- 案例展示页
- 其他类型

不同类型可能使用不同的设计语言，要针对性学习。

## 第二步：视觉拆解（8 个维度）

拿到参考网页后，不要直接写代码。先按下面 8 个维度逐一拆解。

### 2.1 页面节奏

观察并记录：
- 是长滚动连续阅读，还是分段式布局？
- 每个模块是强对比还是温和递进？
- 首屏是先打气场，还是先讲功能？
- 模块之间有没有"呼吸点"（留白段、分隔符、颜色切换）？

**提炼要点**：页面的整体节奏感是什么？快速切换还是舒缓流动？

### 2.2 信息层级

观察并记录：
- 主标题多大、多紧、多重？
- 小标签怎么辅助结构（英文标签、图标、编号）？
- 正文和数据谁更抢眼？
- 页面里到底有几层视觉权重？（通常 3-4 层最佳）

**提炼要点**：信息的优先级是如何通过视觉传达的？

### 2.3 卡片系统

观察并记录：
- 圆角尺寸是什么？（小圆角 4-8px 偏理性，大圆角 12-16px 偏柔和）
- 边框是否存在，粗细和透明度如何？
- 阴影深浅怎样？（无阴影、微阴影、重阴影各有什么气质）
- 卡片内部排版顺序（图标在上还是在左？标题和描述的间距）
- 是否有统一的 badge、chip、metric、footer

**提炼要点**：卡片的视觉语言是什么？如何建立统一感？

### 2.4 留白与网格

观察并记录：
- 模块之间的垂直呼吸感（40px? 60px? 80px?）
- 卡片之间的横向密度
- 内容是否集中在一条主轴上
- 有没有通过对齐建立秩序感
- 最大内容宽度是多少？

**提炼要点**：留白如何创造呼吸感？网格如何建立秩序？

### 2.5 色彩策略

观察并记录：
- 大面积底色是什么？（白、灰、深色）
- 彩色用在什么地方：字、标签、边框、背景块还是强调词？
- 是高饱和驱动，还是低饱和层次驱动？
- 有几种主色？是互补配色还是同色系递进？

**提炼要点**：色彩如何建立层次和情绪？

### 2.6 装饰元素

观察并记录：
- 有无点阵、网格线、渐变雾、细线、胶囊标签等装饰？
- 装饰是铺满页面，还是只做局部点缀？
- 装饰有没有抢走内容注意力？
- 哪些装饰值得借鉴，哪些应该去掉？

**提炼要点**：装饰如何增强氛围而不喧宾夺主？

### 2.7 排版细节

观察并记录：
- 标题用什么字族？衬线还是无衬线？
- 英文和中文是否用了不同字体？
- 行高是紧凑（1.4）还是舒适（1.7-1.85）？
- 有没有用 letter-spacing 做小标签的精致感？

**提炼要点**：排版如何传达专业感和可读性？

### 2.8 总结提炼

至少回答以下 4 个问题：

1. **它为什么显得高级？**（找到核心原因，不是笼统说"好看"）
2. **它最值得借鉴的结构语言是什么？**
3. **它最不该照抄的部分是什么？**
4. **准备如何用本系统的设计语言做出自己的版本？**

## 第三步：定义原创方向

根据拆解结果和用户目标，定义一个原创方向。

### 3.1 命名原创方向

给你的设计方向起一个名字，例如：
- Editorial OS（编辑系统）
- Data Canvas（数据画布）
- Workflow Gallery（工作流画廊）
- Command Center（指挥中心风格）
- Minimal Flow（极简流动式）

### 3.2 明确设计决策

必须明确以下内容：

**页面气质**：
- 商务专业 / 活力创意 / 简约优雅 / 科技未来 / 温暖人文

**模块结构**：
- 采用哪种布局模式（Editorial / Dashboard / Single）
- 包含哪些核心模块
- 模块的排列顺序

**配色方向**：
- 使用哪些马卡龙色（2-3 种主色）
- 底色选择（纯白 / 浅灰 / 彩色浅底）
- 彩色的使用策略（点缀 / 面层 / 强调）

**与参考页的关键差异**：
- 借鉴了什么（具体说明）
- 创新了什么（具体说明）
- 为什么这样做更好

### 3.3 快速推进原则

当用户只想先看效果时，可以直接推荐一个最强方向并推进，不必拉长讨论。

推荐话术：
- "我建议采用 [方向名] 风格，[一句话说明理由]，现在开始制作，做好后你可以提出调整意见。"

## 第四步：原创实现

### 4.1 使用本系统的组件

从 `design-system.md` 中选择合适的组件，不要试图复制参考页的 CSS。

**核心原则**：
- 用本系统的卡片语言（card-light + 马卡龙色）
- 用本系统的排版规范（Noto Sans SC / Noto Serif SC）
- 用本系统的间距体系（40px / 48px / 60px 模块间距）

### 4.2 内容处理

**如果用户提供了具体内容**：
- 使用用户的内容，但重新组织结构
- 不要照搬参考页的标题和文案

**如果用户没有提供内容**：
- 使用合理的随机文案或示例内容
- 确保视觉上像真实成品

### 4.3 页面结构

至少包含：
- 强首屏（3 秒内能看出主题）
- 2-4 个节奏清晰的主模块
- 统一的卡片语言
- 响应式布局

## 第五步：自检与交付

### 5.1 视觉自检

参考 `quality-checklist.md` 进行全面检查，重点关注：
- 首屏冲击力
- 卡片统一性
- 色彩干净度
- 打印友好度

### 5.2 对比检查

将生成的页面与参考页对比：
- 是否保留了参考页的精髓？
- 是否避免了直接抄袭？
- 是否有自己的设计语言？
- 视觉品质是否达到或超越参考页？

### 5.3 交付说明

交付时说明：
- 采用了什么原创方向
- 从参考页借鉴了什么
- 创新了什么
- 文件保存位置
- 1-2 个可选的优化方向

## 常见陷阱与避免方法

### 陷阱 1：照抄结构

**表现**：页面的模块顺序、标题层级、卡片布局与参考页完全一致

**避免方法**：
- 先提炼参考页的"为什么"（为什么这样设计）
- 再用本系统的组件实现"怎么做"
- 模块顺序可以调整，标题可以重写

### 陷阱 2：复制配色

**表现**：直接使用参考页的颜色值

**避免方法**：
- 只学习参考页的配色策略（如"彩色只做点缀"）
- 使用本系统的马卡龙六色
- 根据用户的风格偏好选择 2-3 种主色

### 陷阱 3：忽略统一性

**表现**：每个模块的卡片样式都不一样，像拼凑的

**避免方法**：
- 全页使用统一的 card-light 基础样式
- 圆角、阴影、边框保持一致
- badge、icon-box 等组件样式统一

### 陷阱 4：过度装饰

**表现**：添加了大量参考页没有的装饰元素

**避免方法**：
- 装饰要克制，只在必要时使用
- bg-dot 点阵背景只用于封面或大留白区
- 渐变文字每页至多一个

## 成功案例模式

### 模式 A：节奏借鉴

**参考页特点**：首屏大气 → 数据展示 → 功能介绍 → 案例验证 → 联系方式

**借鉴策略**：
- 保留这个节奏顺序（因为符合用户认知）
- 但每个模块用本系统的组件重新设计
- 配色、字体、卡片样式完全不同

### 模式 B：卡片语言借鉴

**参考页特点**：所有信息都用统一的圆角卡片承载，卡片内有图标+标题+描述

**借鉴策略**：
- 学习"卡片统一性"这个理念
- 用本系统的 card-light + icon-box 实现
- 圆角、阴影、间距按本系统规范

### 模式 B：留白策略借鉴

**参考页特点**：模块之间有大量留白，不拥挤

**借鉴策略**：
- 学习"呼吸感"这个理念
- 模块间距设置为 60-80px
- 内容最大宽度限制在 1000px

## 快速决策表

| 参考页特征 | 借鉴什么 | 创新什么 | 使用组件 |
|---------|--------|--------|--------|
| 分屏式强节奏 | 一屏一主题的叙事节奏 | 配色、卡片样式 | Editorial 模式 + card-light |
| 数据密集型 | 信息组织方式 | 视觉层次、配色 | Dashboard 模式 + metric-card |
| 长文章式 | 阅读节奏、模块分界 | 排版细节、强调方式 | Editorial 模式 + editorial-quote |
| 极简风格 | 留白策略、克制配色 | 增加微妙层次感 | Single 模式 + 极简配色 |
| 卡片统一 | 统一性理念 | 卡片内部结构 | card-light + icon-box |

## 总结

参考学习的核心是：
1. **深度拆解**：理解参考页为什么好
2. **提炼精髓**：找到可借鉴的设计理念
3. **原创实现**：用本系统的语言重新表达
4. **避免抄袭**：结构、配色、文案都要创新

记住：好的参考学习是"站在巨人的肩膀上"，而不是"穿巨人的衣服"。
