# 质量验收清单

完成 HTML 后，必须按以下清单逐项自检。

## A. 首屏冲击力

- [ ] 3 秒内能看出页面主题
- [ ] 主标题有气场，不像普通后台标题
- [ ] 不是一上来就堆满信息
- [ ] 首屏有明确的视觉焦点（品牌图标、渐变标题、大数字等）

**不合格表现**：
- 首屏只有一行小标题
- 信息密度过高，看不出重点
- 标题字体太小或太普通

**修复方法**：
- 标题至少 32px，重要页面用 44px
- 使用 Noto Serif SC 字体，font-weight 900
- 标题中的关键词用渐变色（gt-* 类）
- 添加品牌 icon 或装饰元素

## B. 页面节奏

- [ ] 每个主模块有明显边界（间距、颜色变化、分隔线）
- [ ] 模块顺序有讲故事的感觉
- [ ] 不会从头到尾都是同一种卡片密度
- [ ] 如果超过 5 个模块，中间有至少一次节奏变化（深色段、数据段、公式段等）

**不合格表现**：
- 所有模块看起来一样，没有起伏
- 模块之间挤在一起，没有呼吸感
- 从头到尾都是白底 + 卡片，单调

**修复方法**：
- 模块间距至少 60px
- 每 3-4 个模块后插入一个节奏变化点
- 可以用深色段（dark-section）、大数据展示（stats）、公式图（journey-formula）来打破节奏

## B2. 信息密度（v1.3.0 新增，必检）

这一项用于对抗「稀疏空壳」问题。所有生成页面必须通过。

- [ ] 单张卡片至少承载 3 条有效信息（图标 + 标题 + 描述，或 数值 + 标签 + 变化率）
- [ ] 卡片高度由内容决定，没有被垂直空白撑高
- [ ] grid gap ≤ card padding（避免密度倒挂）
- [ ] card-light padding 在 18-20px 区间，没有写成 22px+
- [ ] section-head 不超过 100px 垂直占位
- [ ] Hero 不超过 480px 高度
- [ ] 首屏 1080px 视口内能看到 Hero + 半个内容模块

**不合格表现**：
- 单张卡片只装了一个图标和两行字，高度却有 260px+
- 卡片之间距离比卡片内部还松（gap 16px vs padding 22px）
- 章节开头占 180px 垂直空间只为了放一个小标签和一个标题
- 首屏只能看见 Hero 一个东西
- section 外壳是一个大卡片（padding 36px），里面又嵌 card-light（padding 22px）
- section-divider 上下有 16px+ margin，显得像独立元素而不是标题的附属

**修复方法**：
- 卡片内容少时：增加信息（补充 meta、数据、次要描述），或减小 padding 到 14-16px
- 卡片内容多时：padding 保持 18-20px，不要拉到 24px+
- grid-4 用 gap 12px，grid-3 用 gap 14px，grid-2 用 gap 16px
- section-head 压缩：eyebrow 和 h2 之间 4-8px，h2 和 divider 之间 4-6px
- 去掉 section 外壳卡片：直接让 section 躺在页面底色上，只用 border-bottom 分隔

## C. 卡片统一性

- [ ] 圆角体系一致（不会一张 8px 一张 20px）
- [ ] 阴影深浅一致
- [ ] badge / chip / metric 的语言统一
- [ ] 没有某一张卡突然像另一套设计系统
- [ ] 卡片内部的图标容器大小和样式统一

**不合格表现**：
- 有的卡片圆角 10px，有的 20px
- 有的卡片有阴影，有的没有
- icon-box 大小不一致

**修复方法**：
- 全页统一使用 card-light 基础样式
- 圆角统一为 14px（大卡）或 12px（中卡）
- 所有 icon-box 统一为 40px × 40px，圆角 10px
- badge 统一为 2px 10px padding，圆角 100px

## D. 色彩干净度

- [ ] 白底保持干净，不被大面积高饱和色块污染
- [ ] 彩色只做层次和强调，不铺满页面
- [ ] 马卡龙色够柔和，不脏不灰
- [ ] 颜色搭配不超过 3 种主色 + 1 种辅色
- [ ] 深色段（如果有）的配色独立完整

**不合格表现**：
- 白底页面出现大片高饱和色块
- 使用了马卡龙色系之外的颜色
- 颜色种类过多，超过 4 种主色
- 紫色渐变配白底（典型 AI 味）

**修复方法**：
- 彩色只用在：badge、icon-box 背景、渐变文字、section-divider
- 卡片底色用极浅底（-50）或浅底（-100），不用主色（-500）
- 商务风格优先用天空蓝/薄荷绿/奶油黄
- 避免使用薰衣草紫作为大面积主色

## E. 排版质量

- [ ] 使用了 Noto Sans SC / Noto Serif SC，不是默认字体
- [ ] 标题大小有明确的层级差异（不是 28/26/24 这种暧昧差距）
- [ ] 英文小标签有 letter-spacing 和 uppercase 处理
- [ ] 正文行高不低于 1.7
- [ ] 没有出现破折号（——）

**不合格表现**：
- 使用了 Arial、Inter、Roboto 等通用字体
- 标题层级不明显，h1 和 h2 只差 2-4px
- 英文标签没有大写和字间距处理
- 正文行高太紧，小于 1.6

**修复方法**：
- 引入 Google Fonts：Noto Sans SC、Noto Serif SC、DM Sans
- 标题层级：h1 (44px) → h2 (32px) → h3 (22px) → h4 (15px)
- 英文标签：font-size: 12px; letter-spacing: 0.22em; text-transform: uppercase;
- 正文：line-height: 1.75 或 1.8

## F. 打印友好度

- [ ] 所有内容默认可见，不需要点击/滚动/等待动画
- [ ] 没有任何 CSS animation / transition / hover 效果
- [ ] 打印预览无空白页、无内容缺失
- [ ] 有 @media print 样式
- [ ] 深色段在打印时可读（如果有）

**不合格表现**：
- 使用了 animation 或 transition
- 有 hover 效果
- 打印时出现空白页
- 深色段在打印时文字看不清

**修复方法**：
- 删除所有 animation、transition、transform 动效
- 删除所有 :hover 伪类
- 添加 @media print 样式：
  ```css
  @media print {
      body { background: #fff; }
      .card-light { break-inside: avoid; }
      .page-number { display: none; }
  }
  ```

## G. 响应式基础

- [ ] 桌面端首屏构图完整
- [ ] 平板端（1024px 以下）多列网格回落到 2 列
- [ ] 手机端（640px 以下）回落到 1 列
- [ ] 标题不炸行、不挤压
- [ ] 有 @media 断点处理

**不合格表现**：
- 在小屏幕上横向滚动
- 多列网格在小屏幕上挤在一起
- 标题在小屏幕上换行不自然

**修复方法**：
- 添加响应式断点：
  ```css
  @media (max-width: 1024px) {
      .grid-3, .grid-4, .grid-5 { grid-template-columns: 1fr 1fr !important; }
  }
  @media (max-width: 640px) {
      .grid-2, .grid-3, .grid-4, .grid-5 { grid-template-columns: 1fr !important; }
  }
  ```

## H. 代码质量

- [ ] 单文件 HTML，CSS 全部在 `<style>` 中
- [ ] 外部依赖仅有 Google Fonts + Font Awesome
- [ ] 无 JavaScript
- [ ] HTML 语义正确（section / h1-h4 / p 使用合理）
- [ ] 无浏览器控制台报错

**不合格表现**：
- CSS 写在外部文件
- 引入了 jQuery 或其他 JS 库
- 使用了大量 div，没有语义化标签
- 控制台有 404 错误或警告

**修复方法**：
- 所有 CSS 内联在 `<style>` 标签中
- 删除所有 JavaScript
- 使用语义化标签：section、article、header、footer、h1-h4
- 检查所有外部资源链接是否正确

## I. 报告页专项检查

（仅当输出为报告类页面时检查）

- [ ] 首屏能看出这是"报告"而不是普通营销页
- [ ] 有独立的 Executive Summary / Highlights 模块
- [ ] 关键结论被做成卡片、指标或摘要，不埋在正文里
- [ ] 建议 / 风险 / 下一步至少有一个单独模块承载
- [ ] 模块顺序符合汇报逻辑：结论先于细节

**不合格表现**：
- 看起来像产品介绍页，不像报告
- 没有摘要或关键结论模块
- 重要信息埋在大段正文中
- 模块顺序混乱，先讲细节再讲结论

**修复方法**：
- 首屏添加"报告"标识（英文标签：REPORT）
- 第二屏必须是 Executive Summary 或 Key Highlights
- 关键数据用 stats-grid 展示
- 建议/风险用独立的 card-light 卡片承载
- 模块顺序：封面 → 摘要 → 关键发现 → 详细分析 → 建议 → 下一步

## 评分参考

**满足以上全部检查项** → 可直接交付

**满足 80% 以上** → 微调后交付

**低于 80%** → 需要返工

## 重点关注的 6 个致命项

以下 6 项不满足则必须修复：

1. **首屏没有冲击力** → 标题太小或太普通
2. **卡片体系不统一** → 圆角、阴影、样式不一致
3. **有动画或交互效果** → 违反静态输出原则
4. **打印时出现空白或内容缺失** → 影响实际使用
5. **使用了默认字体** → 视觉品质不达标
6. **信息密度不足**（v1.3.0 新增）→ 卡片承载信息量少、空白撑高高度、密度倒挂

## 自检流程

1. **视觉检查**（5 分钟）
   - 在浏览器中打开 HTML
   - 从上到下滚动一遍
   - 检查首屏、节奏、卡片、配色

2. **功能检查**（3 分钟）
   - 打印预览（Cmd/Ctrl + P）
   - 调整浏览器窗口大小（测试响应式）
   - 打开浏览器控制台（检查错误）

3. **代码检查**（2 分钟）
   - 搜索 "animation"、"transition"、"hover"
   - 搜索 "Inter"、"Roboto"、"Arial"
   - 搜索 "——"（破折号）
   - 搜索 "padding: 2[2-9]" 或 "padding: 3" 检查是否有过大 padding
   - 搜索 "gap: 2" 检查是否有 gap ≥ 20px
   - 检查是否有外部 JS 文件

4. **对比检查**（如果有参考页）
   - 视觉品质是否达到或超越参考页
   - 是否避免了直接抄袭
   - 是否有自己的设计语言

## 常见问题与修复

### 问题 1：首屏太平淡

**症状**：首屏只有一行标题，没有视觉冲击力

**修复**：
- 添加 bg-dot 点阵背景
- 标题加大到 44px，使用 Noto Serif SC
- 添加品牌 icon（cover-brand）
- 添加英文标签（cover-tag）
- 添加渐变强调词（gt-*）

### 问题 2：卡片不统一

**症状**：每张卡片的圆角、阴影都不一样

**修复**：
- 全局搜索 "border-radius"，统一改为 14px 或 12px
- 全局搜索 "box-shadow"，统一使用 var(--card-shadow)
- 确保所有卡片都有 card-light 类

### 问题 3：颜色太多

**症状**：页面使用了 5-6 种不同的颜色

**修复**：
- 选择 2-3 种主色（如天空蓝 + 薄荷绿）
- 其他颜色只在必要时使用
- 删除不在马卡龙六色中的颜色

### 问题 4：打印空白

**症状**：打印预览时出现空白页

**修复**：
- 检查是否有 animation 或 opacity: 0
- 添加 @media print 样式
- 确保所有内容默认可见

### 问题 5：移动端错乱

**症状**：在小屏幕上布局错乱

**修复**：
- 添加 viewport meta 标签
- 添加响应式断点
- 多列网格在小屏幕上改为单列

## 交付前最后检查

在交付给用户前，最后检查：

1. [ ] 文件已保存到正确位置（桌面 claudecode 输出文件夹）
2. [ ] 文件名清晰易懂（如 product-intro.html）
3. [ ] 在浏览器中打开确认无误
4. [ ] 打印预览确认无空白
5. [ ] 准备好交付说明（布局模式、核心模块、优化方向）

## 总结

质量验收的核心是：
- **首屏有冲击力**
- **卡片体系统一**
- **色彩干净克制**
- **打印友好**
- **代码规范**

记住：宁可多花 2 分钟自检，也不要交付有明显问题的页面。
