# HTML Beautifier（多功能 HTML 美化系统）

一套从真实商业产品中提炼的 HTML 视觉设计系统，能将任何输入内容转换为专业级视觉化页面。
整合了幻灯片、长滚动、看板、单页等多种布局模式，所有输出均为纯静态单文件 HTML，
零动画零交互，打开即可打印、截图或直接作为汇报附件使用。

## 核心设计理念

「卡片即信息容器，留白即呼吸感，节奏即叙事力」

所有内容通过精心设计的卡片、标签、图标组合来承载，不堆砌文字。
每一个模块都是一张完整的视觉海报，有清晰的信息层次。
页面整体有讲故事的节奏感，从首屏的气场到收尾的行动号召，一气呵成。

## 使用流程（重要）

### 第一步：读取输入内容

支持多种输入方式：
1. 用户直接提供文本内容
2. 读取用户指定的文件（.txt, .md, .csv, .json 等）
3. 参考现有网页（URL 或本地 HTML）进行学习和创新

### 第二步：与用户确认风格和场景

在开始制作前，必须通过 AskUserQuestion 工具询问用户：

**问题 1：使用场景是什么？**
选项：
- 方案展示/产品介绍（适合 Deck 模式）
- 数据报告/分析总结（适合 Editorial 或 Dashboard）
- 数据看板/状态汇总（适合 Dashboard）
- 单页海报/简历/邀请函（适合 Single）
- 参考学习某网页，做创新版本（需要额外输入参考 URL）

**问题 2：希望的视觉风格？**
选项：
- 商务专业（克制配色，天空蓝/薄荷绿为主）
- 活力创意（丰富配色，使用全部马卡龙色系）
- 简约优雅（极简设计，大量留白）
- 让 AI 根据内容自动选择

**问题 3：是否需要参考现有网页？**
选项：
- 不需要，直接生成
- 需要，我会提供参考 URL 或本地 HTML 文件

### 第三步：选择布局模式

根据用户的场景选择和内容特征，自动选择最合适的布局模式：

#### 模式 A：Deck（幻灯片翻页式）
**适用于**：方案展示、pitch deck、公司介绍、产品介绍
**特征**：scroll-snap 全屏翻页，每页一屏，页码标注
**最少页数**：3 页（封面 + 内容 + 结束），不限上限

#### 模式 B：Editorial（编辑式长滚动）
**适用于**：分析报告、复盘总结、深度内容、研究文档、文章型内容
**特征**：连续滚动阅读，模块间有明确分界，无翻页
**适合**：内容量大但不需要一屏一屏展示的场景

#### 模式 C：Dashboard（看板/仪表盘式）
**适用于**：数据总览、项目状态、多维信息汇总、对比分析、经营摘要
**特征**：单屏或少量滚动，信息密度高，卡片网格排列
**适合**：需要一眼看全貌的场景

#### 模式 D：Single（单页海报式）
**适用于**：活动海报、单页简历、产品单页、邀请函
**特征**：一页搞定，居中排版，强视觉冲击
**适合**：信息量少但要求精致的场景

### 第四步：生成 HTML

1. 读取 `references/design-system.md` 获取完整 CSS 组件代码
2. 如果是参考学习场景，读取 `references/visual-learning-guide.md`
3. 根据选定的布局模式和风格，组装页面
4. 进行视觉自检（参考 `references/quality-checklist.md`）
5. 输出到桌面 claudecode 输出文件夹

### 第五步：交付说明

交付时简要说明：
- 采用了哪种布局模式，为什么
- 页面包含哪些核心模块
- 文件保存位置
- 1-2 个可选的优化方向（如果用户需要调整）

## 四种布局模式详解

### 布局模式选择逻辑

1. 有明确的章节/步骤/时间线叙事 → Deck
2. 长篇分析、内容量大、需要连续阅读 → Editorial
3. 多维数据并列、状态汇总、对比表格 → Dashboard
4. 信息量少、一页足够、强调视觉 → Single
5. 用户说"像 PPT 一样" → Deck
6. 用户说"看板""dashboard" → Dashboard
7. 用户说"报告""分析""汇报""复盘" → Editorial；如果报告以数据总览为主则选 Dashboard
8. 不确定时，默认 Deck（最通用）

### Deck 模式标准结构

| 页码 | 模板         | 内容               | 何时使用              |
|----|------------|------------------|-------------------|
| 1  | Cover 封面   | 品牌 + 标语 + 副标题  | 必须有               |
| 2  | About 关于   | 团队介绍 + 时间线     | 有团队/历史信息时        |
| 3  | Challenge  | 行业痛点 2x2 宫格    | 需要说明问题背景时        |
| 4  | Solution   | 三层架构纵向堆叠      | 展示解决方案/产品架构时     |
| 5  | Stats 数据   | 关键数据 + 能力亮点   | 有数据指标时           |
| 6  | Capabilities | 功能矩阵 5+3 列    | 展示多个功能/特性时       |
| 7  | Catalog 目录 | 场景列表分组展示      | 有多个场景/案例时        |
| 8  | Journey 旅程 | 合作路径 / 进化路径   | 展示流程/路线图时        |
| 9  | Proven 验证  | 深色背景实绩展示      | 有成功案例/里程碑时       |
| 10 | CTA 联系    | 结束页 + 联系方式     | 必须有               |

### Dashboard 模式骨架

- 顶部：页面标题 + 状态概览条
- 主体：2-4 列卡片网格
- 底部：汇总栏或行动项

### Editorial 模式骨架

- Hero 首屏（标题 + 摘要）
- 2-5 个内容模块（各有章节标题 + 正文/卡片）
- 收口总结或 CTA

## 参考学习模式（可选）

当用户提供参考网页时，遵循以下流程：

### 1. 视觉拆解

读取 `references/visual-learning-guide.md`，按清单分析参考页：
- 页面节奏（翻页式 vs 连续滚动）
- 信息层级（标题大小、权重分布）
- 卡片系统（圆角、边框、阴影）
- 留白与网格（模块间距、对齐方式）
- 色彩策略（底色、彩色用法）
- 装饰元素（点阵、渐变、细线）

### 2. 定义原创方向

不抄袭参考页的结构和文案，而是提炼精髓后用本系统的设计语言重新构建。
明确说明：
- 借鉴了什么（节奏、卡片语言、留白策略）
- 创新了什么（配色、模块结构、信息组织）

## 设计规范速览

完整规范见 `references/design-system.md`，以下为关键要点：

### 配色：马卡龙六色体系

每色三级：主色（-500）、浅底（-100）、极浅底（-50）

| 色名   | 主色       | 用途         |
|--------|-----------|-------------|
| 玫瑰粉 | #e8889a   | 强调、警示    |
| 薄荷绿 | #6bc4a6   | 正向、完成    |
| 薰衣草 | #a78bda   | 品牌、引导    |
| 奶油黄 | #e4c56a   | 提醒、金色调   |
| 天空蓝 | #7ab4d9   | 信息、数据    |
| 蜜桃橙 | #e8a87c   | 温暖、活力    |

**配色原则**：
- 白底页面中，彩色只做点缀和浅面层，不做大片高饱和色块
- 商务风格优先使用天空蓝/薄荷绿/奶油黄组合
- 活力风格可使用全部六色，但每页主色不超过 3 种

### 字体

- 中文正文：Noto Sans SC（300-900）
- 中文标题：Noto Serif SC（600-900）
- 英文/数据：DM Sans
- 图标：Font Awesome 6.5

### 六大核心组件

1. **card-light**：白底 + 淡边框 + 双层微阴影，万能信息容器
2. **gt-渐变文字**：标题强调词用渐变色，每页至多一个
3. **section-divider**：章节标题下方 48px 渐变短线
4. **badge**：药丸标签，分类标识和状态标记
5. **num-dot**：编号圆点，用于列表和步骤序号
6. **icon-box**：方形圆角图标框，承载 Font Awesome 图标

## 严格禁止

- ❌ 任何 CSS 动画（animation / transition / transform 动效）
- ❌ 任何 JavaScript 交互逻辑
- ❌ hover 效果（打印时不可见，视觉不一致）
- ❌ Inter、Roboto、Arial 等通用字体
- ❌ 紫色渐变配白底的「AI 味」配色
- ❌ 标题中使用破折号（——）
- ❌ 使用 emoji，统一用 Font Awesome 图标
- ❌ 大段文字堆砌，所有信息必须通过卡片结构承载
- ❌ reveal 动效导致截图或打印出现空白

## 输出规范

- 单文件 HTML，所有 CSS 内联在 `<style>` 中
- 外部依赖仅限 Google Fonts CDN + Font Awesome CDN
- 不依赖任何 JavaScript 框架或库
- 所有内容默认可见，无需用户操作
- 文件默认保存到桌面 claudecode 输出文件夹
- 包含 `@media print` 样式，确保打印效果正确
- 包含基础响应式样式（`@media max-width: 1024px`）

## 质量验收

完成 HTML 后，按 `references/quality-checklist.md` 中的清单自检：
- 首屏 3 秒内能看出页面主题
- 卡片体系统一（圆角、阴影、标签语言一致）
- 白底干净，彩色只做层次点缀
- 模块间有明确边界和节奏感
- 打印预览无空白、无缺失
